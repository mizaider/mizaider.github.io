Описание панели лицензирования

Что умеет панель:​

  1. Структурировать список лицензий с возможность поиска по списку.
  2. Возможность указания срока действия лицензии.
  3. Возможность задать триальное время работы продукта.
  4. Блокировка использования скрипта на домене одной кнопкой.
  5. Логирование доменов на которых скрипт был запущен без покупки.
  6. Автоматическая рассылка абуз владельцам доменов и хостерам сайтов из пункта 5.
  7. Добавление и мониторинг нескольких продуктов.
  8. Мониторинг целостности файлов нескольких версий одного продукта.
  9. Автоматическое генерирование готового кода проверки лицензии двух видов - С полем ввода ключа и Просто код проверки.
  10. Шифрование ответа API сервера лицензий.
  11. API для работы с панелью.
  12. Использование RSA для подписи.
  13. Сильный PHP обфускатор.
  14. Онлайн кодирование IonCube 9.
  15. Дешифратор меток файлов.
  16. Выдача lic ключей которые можно использовать как способ валидации лицензии.
  17. Полная автоматизация выдачи лицензий.
  18. Общий чёрный список доменов, почт и логинов любителей вареза, который ведут сами разработчики.

Главная страница:


  1. Меню навигации по разделам
  2. Информация о пользователе и уровне доступа, а так же о новых сообщениях и уведомлениях на форуме
  3. Общая статистика системы в реальном времени
  4. Новости панели

Теперь рассмотрим каждый раздел более подробно:

Раздел управления лицензиями


Список лицензий:
  1. Список лицензий
  2. Добавление лицензии
  3. Добавления ключа активации
  4. Общий id лицензии в системе
  5. id лицензии для управления через api
  6. Домен или домены на которых разрешено запускать продукт
  7. Последний ip с которого происходила проверка состояния лицензии
  8. Почта клиента
  9. Место покупки лицензии
  10. Номер чека
  11. Активационный ключ
  12. Дата добавления лицензии в систему
  13. Дата истечения лицензии
  14. Продукт к которому прикреплена лицензия
  15. Статус лицензии. На скриншоте лицензия в ожидании ввода ключа клиентом.
  16. Комментарий

Создание лицензии:
  1. Домен для которого выдаётся лицензия
  2. Почта клиента
  3. Место покупки лицензии (Может быть селект если указать в настройках перечень сайтов где продаётся продукт)
  4. Номер чека присвоенный сайтом продавцом при продаже продукта
  5. Срок действия лицензии
  6. Селект для выбора продукта для которого выдаётся лицензия
  7. Селект для выбора статуса лицензии
  8. Ключ лицензии который сгенерирован по маске в настройках
  9. Комментарий к лицензии
  10. Дополнительные параметры которые могут быть переданы в ответе api

Создание ключа лицензии:
  1. Продукт к которому создаётся ключ
  2. Сам ключ

Раздел управления продуктами


Список продуктов:
  1. Список продуктов
  2. Список версий продуктов
  3. Новый продукт
  4. Новая версия продукта
  1. Общий id продукта в системе
  2. id продукта для управления им через api
  3. Полное имя продукта (Отображается в активационных письмах)
  4. Короткое имя продукта (Отображается в списках)
  5. Дата добавления продукта
  6. Цена
  7. Общее количество лицензий к продукту в системе
  8. Цена*Количество
  9. Песочница (При любом статусе лицензии отдаёт код 203 - Продукт в режиме песочницы)
  10. Триал. Время которое продукт работает без покупки. Домен появляется в базе.
  11. Управление продуктом. Редактировать. Удалить.

Список версий продуктов:
  1. Общий id версии в системе
  2. id для управления версией с помощью api
  3. Идентификатор версии передаётся при проверке лицензии на продукт
  4. Дата добавления
  5. Продукт к которому принадлежит версия
  6. Хеши контролируемых файлов для проверки их целостности
  7. Действия. Удалить

Создание продукта:
  1. ID продукта на сайте liveopencart.ru
  2. ID продукта на сайте opencartforum.com
  3. ID продукта на сайте forum.devsaid.com
  4. Полное имя продукта
  5. Цена
  6. Короткое имя продукта
  7. Режим песочницы, при включении в любом случае api отвечает 20* кодом
  8. Триальный режим. Возможность работы скрипта без оплаты на срок указанный тут, далее блокируется

Создание версии продукта:
  1. .zip архив с файлами целостность которых нужно контролировать
  2. После нажатия нужно будет указать идентификатор версии и продукт.

Раздел обфускации кода и генерирования скриптов защиты


Раздел генерирования проверяющего скрипта для продукта:
Выбор продукта для которого генерируется скрипт и тип скрипта проверки:
  1. Генерирование скрипта проверки
  2. Обфускатор и онлайн кодирование IonCube 9
  3. Проверка lic файлов
  4. Декодер меток форума
  5. Выбор продукта
  6. Идентификатор версии для контроля целостности файлов
  7. Тип скрипта проверки

Раздел обфускации кода:
  1. Исходный код для обфускации
  2. Перемешивание строк. Код приобретает вид PHP: goto RIxzJFL9ViAqL8Zv; bENHtaCGsiTMBPuC: define("\x43\122\x59\120\x54\x5f\x52\x53\101\x5f\115\117\x44\105\x5f\111\x4e\124\105\122\x4e\101\x4c", 1); goto ZjP0LGT6lJTfn9r_; h0sX2ZYtppnG34GQ:
  3. Обфускация кода средствами php. После включения появятся поля с выбором профиля обфускации, а так же возможность зафиксировать имена сущностей PHP.
  4. Онлайн кодирование скрипта с помощью IonCube 9. После включения появятся поля с выбором версии PHP, лоадера и другими настройками IonCube.
    5. Можно комбинировать возможности.

Инструмент проверки *.lic ключа на валидность:
  1. Выбираем файл
  2. Жмем проверить и получаем полную информацию по ключу

Инструмент поиска и расшифровки меток файлов:
  1. Вводим текстовую метку
  2. Или выбираем любой файл (из перечисленных расширений)
  3. Нажимаем и получаем строку в виде - ID-логин-почта пользователя который скачал этот файл с форума

Раздел настроек панели


Основные настройки панели:
  1. Маска по которой генерируется ключ при создании лицензии через панель
  2. При включенной опции ответ подписывается приватным ключом RSA
  3. При включенной опции ответ api шифруется ключом который будет введен в поле которое появится при включении опции
  4. При включенной опции содержимое *.lic файла шифруется ключом который будет введен в поле которое появится при включении опции
  5. Селект который укажет панели в каком виде хранить данные в *.lic ключе
  6. Время жизни lic файла
  7. При добавлении домена с неуказанным сроком окончания лицензии она автоматически записывается как time()+ указанное количество дней
  8. Секретный ключ который используется для создания хеша ответа api для проверки подлинности ответа
  9. Если опция включена то в ответе api при валидации домена будут содержаться все данные о лицензии
  10. Если опция включена то при создании лицензии через панель или api на почту которая указана в лицензии как почта клиента уйдет письмо с данными о лицензии и *.lic файлом
  11. Если опция включена то при создании лицензии через пкупку на этом форуме от имени разработчика ухоит личное сообщение покупателю с данными лицензии
  12. Список сайтов через ; для того чтобы в пункте 1.2.3 этого мануала был не инпут а селект для удобства создания лицензий
  13. Ключ для запуска задач по расписанию. (Сбор писем о покупке или рассылка писем пользователям)

Настройки RSA:
  1. Если включено используются Ваши ключи для подписи ответов api и *.lic ключей. Если выключено, наши.
  2. Приватный ключ, может быть введен в любом формате, автоматически отформатируется в строку.
  3. Публичный ключ, может быть введен в любом формате, автоматически отформатируется в строку.

Интеграция с сайтами для получения от них информации о покупке:
  1. Автоматический приём и обработка данных о покупке из писем от сервисов и ресурсов не имеющих api.
  2. URL который нужно добавить в планировщик для запуска процесса сбора писем.
  3. Почтовый ящик на который приходят письма о покупке.
  4. Пароль от этого ящика.
  5. Источник писем о покупке.
  6. Автоматический приём данных о покупке от сайта liveopencart.ru
  7. Ключ для проверки подлинности сгенерированный в кабинете liveopencart.ru
  8. Если включено, при получении данных автоматически будет выдан ключ.
  9. ID пользователя на форуме для привязки аккаунта в панели к аккаунту на форуме
  10. Секретный ключ для гарантии того что информация пришла с форума и для проверки что Вы привязываете своего пользователя а не чужого.

Настройки почтовых уведомлений:

  4. Пароль от него
  5. SMTP порт
  6. Можно указать адрес на который принимать ответы на письма
  7. SSL\TLS

Шаблоны писем:
  1. Использовать свои шаблоны или встроенные
  2. URL который нужно добавить в планировщик для запуска процесса рассылки писем.
  3. Тема письма активации лицензии
  4. Шаблон письма активации лицензии
  5. Тема письма о скором истечении лицензии
  6. Шаблон письма о скором истечении лицензии
  7. Тема письма с жалобой о незаконном использовании продукта
  8. Шаблон письма с жалобой о незаконном использовании продукта
